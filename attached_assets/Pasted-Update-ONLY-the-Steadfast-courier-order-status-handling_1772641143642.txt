--------------------------------
ORDER STATUS TYPES
--------------------------------

Support these statuses:

- Pending
- In Review
- Delivered
- Cancelled

--------------------------------
AUTOMATIC STATUS (API)
--------------------------------

If Steadfast API returns status updates:

Delivered:
- Remove amount from COD Pending
- Add (Sale Amount - Courier Charge) to Cash In Hand
- Deduct courier charge as expense
- Profit = Sale price - product cost - courier charge

Cancelled:
- Remove sale profit
- Courier charge becomes expense
- Sale is marked cancelled

Pending / In Review:
- Amount stays in COD Pending
- Do not add to Cash In Hand yet

--------------------------------
MANUAL STATUS CONTROL
--------------------------------

Allow admin to manually update courier status from the Steadfast page.

Manual options:

Pending
In Review
Delivered
Cancelled

Manual update must trigger the same financial logic as automatic updates.

--------------------------------
COURIER CHARGE HANDLING
--------------------------------

Courier charge must:

Delivered:
Deduct from sale before adding to cash.

Cancelled:
Recorded as expense.
