Fix the financial activity history system.

The Database page must work as a simple activity log.

Every financial action must automatically create a record in the transaction_history table.

Actions that must be recorded:
- Sale
- Purchase
- Expense
- Investment
- Payment Received
- Withdrawal
- Courier Expense
- Return Charge

When any of these actions occur, insert a new record into transaction_history with the following fields:

date
type
reference
description
amount

Example structure:

Date | Type | Reference | Description | Amount

Sale → 
type: Sale

description: product name + customer
amount: sale total

Investment →

amount: investment amount
