TRAINING RULES BASED ON THESE PATTERNS

STEP 1: LINE-BASED PRIORITY

If input has line breaks:

Line 1 (short text, no numbers) → Name
Line containing 11-digit 01 number → Phone
Longest remaining line → Address

---

STEP 2: LABELED PRIORITY (STRICT)

If contains:
নাম:
ঠিকানা:
মোবাইল:
Name:
Phone:
Address:

Always extract by label first.
Ignore position.

---

STEP 3: PHONE PRIORITY (GLOBAL RULE)

Always:

- Detect 11-digit BD number
- Convert Bangla digits to English
- Remove from text before further parsing

---

STEP 4: NAME RULE BASED ON REAL DATA

From examples:

Name characteristics:

- Usually first short line
- 1–3 words
- No digits
- Not long sentence
- Not containing address keywords

If a segment:

- Word count <= 3
- No numbers
- No address keywords
  → Strong Name candidate

---

STEP 5: ADDRESS RULE BASED ON REAL DATA

Address characteristics from examples:

- Often longer text
- Contains location names
- May contain house number (161/7)
- May contain comma
- May contain district name
- May contain abbreviations like ctg

If segment:

- Contains numbers with slash
- Contains comma
- More than 3 words
- Contains district name

→ Treat as Address

---

STEP 6: DISTRICT / LOCATION RECOGNITION

Based on samples include detection for:

cumilla

mugdha
mirpur
hathazari

dhaka

Bangla:

হাজীগঞ্জ
কুমিল্লা
ঢাকা

These increase Address probability.

---

STEP 7: FALLBACK RULE

If structure is:

Single short word
+
Phone
+
Multiple words

→ First short word = Name
→ Rest = Address

---

FINAL GOAL

Using real-world messy examples,
AI must correctly extract:

Name
Phone (English digits only)
Address

Even if:

- Phone at bottom
- Address in middle
- Bangla labels used
- Mixed Bangla & English
- Short abbreviations used

Do NOT change form structure.
Only improve parsing intelligence.
